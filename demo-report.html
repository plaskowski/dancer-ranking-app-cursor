Automation Methods

ðŸš€ What The Real Agent Does

  • Web Automation: Uses Puppeteer to control Chrome and capture Flutter web app screenshots
  • Android Automation: Uses Flutter Driver + ADB to capture Android app and device screenshots
  • Integration Testing: Uses Flutter's integration_test package for cross-platform screenshots
  • Baseline Comparison: Automatically compares new screenshots with baseline images
  • Visual Reports: Generates detailed HTML reports with screenshot galleries
  • Multiple Devices: Supports different screen sizes and device types
  • Custom Scenarios: Follows your defined user interaction flows
  • CI/CD Integration: Works in automated build pipelines

ðŸ”„ Agent Workflow

1. Initialization: Agent checks dependencies, detects devices, creates directories
2. App Startup: Agent starts your Flutter app (web/Android/emulator)
3. Test Execution: Agent follows test scenarios (clicks, scrolls, navigates)
4. Screenshot Capture: Agent takes screenshots at each step programmatically
5. Comparison: Agent compares with baseline images to detect changes
6. Report Generation: Agent creates detailed HTML reports with visual galleries
